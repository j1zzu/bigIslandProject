# Геопортал Большого Уссурийского острова

Локальное приложение Node.js + Leaflet отображает базовую карту OpenStreetMap, WMS/WFS из QGIS Server и тестовые зоны GeoJSON. Карта браузера работает в `EPSG:3857`, рабочий CRS данных QGIS — `EPSG:32653`; GeoJSON и WFS для браузера выдаются в `EPSG:4326`.

## Запуск

Требуется Node.js 18 или новее.

```bash
npm install
copy .env.example .env
npm run dev
```

Команда `npm run dev` запускает сервер и автоматически открывает `http://localhost:3000` в системном браузере. Терминал нужно оставить работающим; для остановки нажмите `Ctrl+C`. Команда `npm start` запускает сервер без автоматического открытия браузера.

Если терминал сообщает, что порт `3000` занят, сайт уже может быть запущен: откройте `http://localhost:3000` либо остановите прежний процесс и повторите команду. По умолчанию используется QGIS Server `192.168.20.21:8080` и проект `world2.qgz`. Параметры можно изменить через `QGIS_SERVER_URL` и `QGIS_PROJECT_PATH` в `.env`; браузер обращается к QGIS только через локальные proxy `/qgis/wms` и `/qgis/wfs`.

## QGIS Server

Проект `big_ussuriysky.qgz` должен использовать `EPSG:32653` и публиковать слои из [qgis/sample-layers.md](qgis/sample-layers.md). Проверка через proxy:

```text
http://localhost:3000/qgis/wms?REQUEST=GetCapabilities
http://localhost:3000/qgis/wfs?REQUEST=GetCapabilities
```

Базовая карта острова загружается Leaflet напрямую из OpenStreetMap и не зависит от QGIS. QGIS WMS используется только как прозрачный дополнительный слой и должен разрешать выдачу в `EPSG:3857`. При росте нагрузки следует добавить MapProxy, GeoWebCache либо Nginx cache/WMTS. WFS предназначен для объектов и атрибутов, а локальный GeoJSON — для небольших тестовых наборов.

## Данные Revit

Прямая работа с `.rvt` запрещена архитектурой проекта. Backend не читает, не загружает и не конвертирует файлы Revit и не зависит от установленного Revit. Единственный поддерживаемый путь: экспорт из Revit в обменный формат, импорт и пространственная проверка в QGIS, затем публикация WMS/WFS или экспорт GeoJSON. Пошаговая инструкция: [qgis/README_QGIS.md](qgis/README_QGIS.md).

## Ограничения

- Базовая карта требует доступ браузера к `tile.openstreetmap.org`; дополнительные WMS/WFS зависят от локального QGIS Server `192.168.20.21`.
- Библиотеки карты загружаются из CDN; для полностью изолированной сети их нужно разместить локально.
- GeoJSON по RFC 7946 хранится в WGS 84 (`EPSG:4326`) с координатами `[longitude, latitude]`.
